Detailed Description

A stage manages actors.

The stage facilitates the interaction of actors in a simulation.

It constructs and owns the actors on stage. It is responsible for maintaining the actors' perceived and collision sets and calls the actors' act methods every tick.

T & kmint::play::stage::build_actor ( Args &&... constructor_arguments)

Constructs an actor.

This is the only way to create an actor that is to be used on stage. This member function accepts an arbitrary number of parameters that are forwarded to the actor's constructor.

Template Parameters
TThe class of the actor. Must be the type of a concrete subclass of actor.
Returns
A reference to the newly created actor. This reference is valid until the actor is removed from play (via actor::remove or stage::remove_actor) or the stage lifetime ends.

◆ remove_actor()

void kmint::play::stage::remove_actor ( actor const & a)

Removes an actor from play.

This function removes the actor from the list of actors on stage. As the stage owns the actor, the actor will be removed from memory and the actor's destructor will be called. Any reference or pointer to that actor is now invalid.

This function invalidates iterators and must not be called from any act method. If the method is called from an act function, an exception is thrown.

Parameters
aActor to remove from play.

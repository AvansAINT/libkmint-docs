Detailed Description

A class that carries information about an event.

event can carry one of a number of events such as the request to end the application (quit_event), a key press (key_down_event). and many others.

What type an event object carries cannot be determined via a traditional getter. Instead, the user must call any of the handle methods to specify how an event must be handled. For instance, the event::handle_quit function accepts a callable object. event::handle_quit will inspect the internal state of the event and if it corresponds to a quit event, it will call its argument with a quit_event object.

Alternatively, the visit method may be called with an object that provides an overloaded call-operator for each of the different event types.

event may be default constructed. In that case, the object holds no event information. None of the handle functions will respond. An event in this state is said to be invalid. This may be checked by calling event::valid.

template<typename Visitor>
void kmint::ui::events::event::visit ( Visitor && f) const
inline

Calls the correct call operator overload on a visitor.

This method should be called with an object that has an overloaded call operator for each event type. visit will inspect the current event and call the correct overload with the corresponding event object.

Example

struct event_handler {
void operator()(quit_event qe) {
// handle quit event
}
void operator()(key_up_event qe) {
// handle key up event
}
void operator()(key_down_event qe) {
// handle key down event
}
// etc.
}
// ...
void process(event e) {
event_handler handler{};
e.visit(handler);
}
